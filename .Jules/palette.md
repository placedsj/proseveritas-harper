## 2025-05-20 - Interactive Cards Accessibility
**Learning:** Clickable cards implemented as `div`s are not keyboard accessible.
**Action:** Use `<button>` elements for interactive cards, ensuring `w-full text-left` classes are applied to maintain layout.
## 2025-05-20 - Global Search Modal Accessibility
**Learning:** Custom modals require explicit keyboard (Escape) and click-outside interaction handling for proper accessibility and UX.
**Action:** Always implement document-level keyboard listeners and event propagation control on modal backdrops.
